Redcoat User Guide

Redcoat is a Node.js application. It is designed to run on a web server so that annotators can access it from their own computers. The installation and setup process is outlined below.

Installing Node.js

The first step is to install Node.js.

  • If you are using Ubuntu, you can install Node.js and npm by using the following commands:

    $ sudo apt-get update
    $ sudo apt-get install nodejs
    $ sudo apt-get install npm
  • If you are using Windows, simply download the Windows installer and follow the instructions. Ensure that npm is also installed (it should be included with the installation by default).
  • If you are using Mac OS X, you can download an installer from the Node.js website. Alternatively, the following command should install both node.js and npm using homebrew:
    $ brew install node

    • For additional installation instructions, please see the Node.js website.

Running the Redcoat Application

Download the latest version of Redcoat here.

Unzip the folder and navigate to the folder's location using command prompt/terminal. You'll then need to install the required node packages using npm:

$ npm install

This may take some time depending on the speed of your internet connection.

Once the packages have been installed, you should be able to run the application using:

$ npm start

Creating the Administrator Account

If this is your first time running the application, you'll be greeted with a message asking for a username and password for the Administrator account. This is a necessary step to ensure that only you are able to set up your project through the browser.

Configuring SMTP Details

You'll also be asked whether you want to send out invitations to annotators using your own STMP server.

If you answer with 'y', you'll be able to enter the host, port, and authentication credentials for your SMTP server. Sending out your annotation project using your own SMTP server will be enabled as an option when setting up your project.

For example, if you have a Gmail account, you can Gmail's SMTP server by entering the following:

SMTP server name: smtp.gmail.com
SMTP username: your Gmail address
SMTP password: your password
SMTP port: 465

If you answer with 'n' (or blank), you can enter your SMTP details in the mailer.js file at any time. Until you do, you won't have the option to send out your annotation project using your own SMTP server. You will still be able to send out the project manually, as Redcoat will provide you with a list of URLs to distribute via email.

Setting up a New Project

Once the server is up and running, you can continue the rest of the setup process in a browser such as Firefox or Chrome.

If you are running the application on your own computer, navigate to localhost:3000 in your browser.

If you are running the application on a web server (and have set up the server using something like ssh), you can (optionally) close the ssh connection and navigate to <your server's IP address>:3000 in your computer's browser. You'll need to ensure that port 3000 is open.

In your browser you can log in using the administrator account details you entered earlier and begin setting up a project.

Required Information

The setup page is fairly self-explanatory, but we have included the required details here.

Section Required Information
Project Details Enter the project name and a brief description of the project. The description will appear in the automated mailouts if you decide to use them.
Dataset Determine whether your data has already been tokenized. There are two options to choose from:
  • "Yes, my data has been tokenized": You'll be asked to upload your tokenized data. The dataset must be saved as a .txt file. Each token within your data must be separated by a newline, and each record must be separated by an additional newline. Click here to download a small example file.
  • "No, my data has not been tokenized": You'll be asked to upload your untokenized data. The dataset must be saved as a .txt file. Each record within your data must be separated by a new line. Click here to download a small example file. Your data will be tokenized using regular expression parsing.
Please note, it's highly recommended that you provide your own tokenized data. Providing untokenized data is only recommended for quickly trying out Redcoat.
Entity Categories Determine how you would prefer to upload the entity categories. There are three options to choose from:
  • "My entity categories are saved in a file": You'll be asked to upload your entity categories. The list of entity categories must be saved as a .txt file. Each line in the file must contain an entity category, followed by a tab character, followed by an abbreviated form of that entity (such as PER for "Person").
    You may optionally add a tag colour by adding an additional tab character and a 6-digit hexadecimal colour code to the end of the line. This will determine the colour of the respective entity's label during annotation.
    Click here to download a small example file.
  • "I'll list the entity categories here": You'll be asked to enter your entity categories into the table that appears on the setup form. The "Abbreviation" will determine what is written in the output file after annotation, while the "Colour" will determine the colour of the tag during annotation.
  • "I'm using the standard 4 class model for Named Entity Recognition": your entity classes will be "Person", "Location", "Organization" and "Misc". This option is provided to serve as a shortcut for users performing NER.
Distribution Determine how you would like to distribute the annotation task. There are four options to choose from:
  • "I'll complete the annotation task myself": Redcoat won't generate any URLs for you to send out to annotators. When submitting the setup form, you'll be able to start annotating right away. You can resume annotation at any time by logging in again using your administrator credentials
  • "I'd like a list of codes that I will send to the annotators myself": You'll be asked to provide the number of annotators that will be working on your project. You will also be asked for the server address (such as your server's IP address or domain name). Redcoat will divide your dataset into batches (1 per annotator) and will generate a list of randomly-generated URLs, such as <your server's address>:3000/annotation/sHd36CJL9z. You can email one of these URLs to each annotator.
  • "I've provided SMTP credentials and want to use my SMTP server to email codes to each annotator": This option will become available if you entered your SMTP credentials when setting up the server (see Configuring SMTP Details). You'll be asked to provide the number of annotators that will be working on your project, and the server address. Redcoat will attempt to use your SMTP server to send out 1 URL (as above) to each annotator via email.

Annotating Data

Once you have set up your project, you (or your annotators) will be able to begin annotating your data.

If you chose the option "I'll complete the annotation task myself" in the Distribution section on the setup page, you'll be directed straight to the annotation page at /annotation. If you selected one of the other options, your annotators will be able to access the annotation pages by visiting their unique URL.

Each annotator's respective dataset is separated into groups of 10 records. One group will appear in the browser at a time, and completing one group of records will save the results to a JSON database and allow users to progress to the next group.

The controls are as follows:

Key Action
Left/right arrow Select previous/next token
Up/down arrow Select previous/next record
Shift + left/right arrow Select multiple tokens at once
1-9, 0 Tag token with the corresponding class
j Jump to group

Statistics Tracking

While the annotation is underway, you can check the results by visiting <your server's address>:3000/results in your browser, and logging in using your administration account. You'll also be able to reach this page via the homepage after a project has been created.

On the results page you'll be able to view how many records have been annotated, along with the number of entities belonging to each class. If you elected to distribute the project to multiple annotators, the page will show progress bars for each annotator indicating how close they are to completion.
